
# Common Network Protocols

> [!info] Overview  
> Protocols are standardized rules that allow devices to communicate over networks.  
> Knowing common protocols is foundational for networking and cybersecurity.

---

# Remote Access & File Transfer

|Protocol|Acronym|Purpose|
|---|---|---|
|Secure Shell|SSH|Secure remote login and command execution|
|File Transfer Protocol|FTP|Transfers files between systems|
|Server Message Block|SMB|File and printer sharing in Windows networks|
|Network File System|NFS|Access files over a network|
|Remote Shell|RSH|Execute commands remotely on Unix systems|

# Wireless & Authentication Protocols

|Protocol|Acronym|Purpose|
|---|---|---|
|Wired Equivalent Privacy|WEP|Older wireless security protocol|
|Wi-Fi Protected Access|WPA|Wireless network protection|
|Temporal Key Integrity Protocol|TKIP|Legacy wireless encryption|
|Extensible Authentication Protocol|EAP|Authentication framework|
|Lightweight EAP|LEAP|Cisco wireless authentication|
|Protected EAP|PEAP|Encrypted authentication tunnel|

---

> [!warning] Wireless Security  
> WEP and TKIP are outdated and insecure by modern standards.

# VPN & Encryption Protocols

|Protocol|Acronym|Purpose|
|---|---|---|
|Virtual Private Network|VPN|Secure encrypted connection|

# Key Takeaways

> [!summary]
> 
> - Protocols define how devices communicate
>     
> - SSH, SMB, HTTP, FTP are core networking protocols
>     
> - OSPF/EIGRP/RIP handle routing
>     
> - VPNs commonly use IPsec + IKE
>     
> - WEP/TKIP are outdated wireless protocols
>     
> - VLANs segment networks logically
>     
> - SNMP manages network devices
>     
> - NAT allows many devices to share one public IP
>
